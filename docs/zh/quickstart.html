快速开始

入门指南

本指南将带您完成使用 Linglong Web 构建生产级异步服务的基本步骤。 我们将涵盖安装、配置、资源管理和服务器启动。

前置要求
  • Python 3.10 或更高版本
  • Redis 服务器（用于缓存、限流、分布式锁）
  • PostgreSQL/MySQL（可选，用于数据库功能）

1. 安装

通过 pip 安装 Linglong Web： 

# 从 PyPI 安装
pip install linglong-web

# 或从 GitHub 安装最新版本
pip install git+https://github.com/10000ms/linglong_web.git

2. 项目结构

典型的 Linglong Web 项目结构如下： 

my-project/
├── app/
│   ├── __init__.py
│   ├── config.py       # 配置类
│   ├── routers/       # API 路由处理器
│   └── main.py        # 应用入口
├── config/
│   └── settings.yaml  # 运行时配置（支持热重载）
├── requirements.txt
└── pyproject.toml

3. 配置

Linglong Web 提供了强大的配置系统，支持热重载。 您可以定义多个配置环境（dev、test、prod）并轻松切换。

基础配置

步骤 1 通过继承 LinglongConfigBase 定义配置类

from linglong_web import LinglongConfigBase

class DevConfig(LinglongConfigBase):
    """开发环境配置"""
    DEBUG = True
    SERVICE_NAME = "my-api"
    
    # 数据库设置
    PGSQL_HOST = "localhost"
    PGSQL_PORT = "5432"
    PGSQL_USER = "postgres"
    PGSQL_PASSWORD = "password"
    PGSQL_DB = "mydb"
    
    # Redis 设置
    REDIS_HOST = "localhost"
    REDIS_PORT = 6379

步骤 2 使用 init_config() 初始化配置

from linglong_web import init_config

# 使用多个环境配置初始化
init_config({
    "dev": DevConfig,
    "prod": ProdConfig,
}, mode_name="dev")

# 或使用环境变量 LINGLONG_CONFIG
# export LINGLONG_CONFIG=prod
# init_config({"dev": DevConfig, "prod": ProdConfig})

热重载配置

对于生产环境，您可以使用支持热重载的 YAML 配置文件， 无需重启服务即可修改配置： 

# config/settings.yaml（此文件可在运行时修改）
# DEBUG: true
# SERVICE_NAME: "my-api"
# REDIS_HOST: "redis.internal"

import yaml
from linglong_web import LinglongConfig

# 从 YAML 文件加载
with open("config/settings.yaml") as f:
    settings = yaml.safe_load(f)

# 应用更新（支持热重载）
LinglongConfig.apply_updates(settings)
方法 描述
init_config(config_dict, mode_name) 使用环境特定的配置类初始化
LinglongConfig.apply_updates(updates) 应用运行时配置更新（支持热重载）
LinglongConfig.snapshot() 获取当前配置值的快照
LinglongConfig.reset() 重置配置到初始状态

4. 资源管理

Linglong Web 为数据库、缓存、消息队列等提供统一的资源管理。 通过单个函数调用初始化所有资源。

初始化资源

from linglong_web import init_resources, Rmanager

# 初始化数据库、Redis 和其他资源
await init_resources(
    db_config={
        "default": {
            "host": "localhost",
            "port": 5432,
            "user": "postgres",
            "password": "password",
            "database": "mydb",
        }
    },
    redis_config={
        "default": {
            "host": "localhost",
            "port": 6379,
        }
    }
)

# 通过 Rmanager 访问资源
db_engine = Rmanager.get_pgsql_engine("default")
redis_client = Rmanager.get_redis("default")
资源类型 访问方法
PostgreSQL Rmanager.get_pgsql_engine(alias)
MySQL Rmanager.get_mysql_engine(alias)
Redis Rmanager.get_redis(alias)
RabbitMQ Rmanager.get_rabbitmq(alias)
MongoDB Rmanager.get_mongo(alias)

关闭资源

from linglong_web import close_resources

# 优雅地关闭所有资源
await close_resources()

5. 构建 API 路由

Linglong Web 提供了简洁优雅的路由系统来定义 API 端点。 

from linglong_web import ServerRouter, build_success_response

router = ServerRouter()

# 使用装饰器定义路由
@router.route("/api/users", method="GET")
async def list_users(request):
    users = [{"id": 1, "name": "Alice"}, {"id": 2, "name": "Bob"}]
    return build_success_response({"users": users})

@router.route("/api/users/{user_id}", method="GET")
async def get_user(request, user_id: int):
    return build_success_response({"id": user_id, "name": f"User {user_id}"})

@router.route("/api/users", method="POST")
async def create_user(request):
    data = await request.json()
    return build_success_response({"user": data}, status=201)

6. 启动服务器

最后，使用您的路由和配置初始化并启动服务器。 

import asyncio
from linglong_web import (
    LinglongAppServer,
    ServerRouter,
    init_config,
    init_resources,
    close_resources,
)

async def main():
    # 1. 初始化配置
    init_config({"dev": DevConfig}, mode_name="dev")
    
    # 2. 初始化资源（数据库、Redis 等）
    await init_resources(db_config={...}, redis_config={...})
    
    # 3. 创建并配置服务器
    server = LinglongAppServer()
    await server.initialize(
        service_name="my-api",
        router=router.get_router(),
        config_dict={"dev": DevConfig},
    )
    
    # 4. 启动服务器
    await server.start(host="0.0.0.0", port=8080)

if __name__ == "__main__":
    asyncio.run(main())

7. 添加装饰器

Linglong Web 提供了强大的装饰器，用于缓存、限流和分布式锁。 

from linglong_web import cacher, limiter, cluster_lock

# 缓存结果 5 分钟
@cacher.cacher(key="user:{user_id}", expire=300)
async def get_user(user_id: int):
    return await db.fetchone("SELECT * FROM users WHERE id = $1", user_id)

# 限流：每分钟 100 次请求
@limiter.limiter(key="api:/users", rate="100/minute")
async def list_users():
    return await db.fetch("SELECT * FROM users")

# 分布式锁：防止并发执行
@cluster_lock.cluster_lock(key="job:sync:{user_id}", timeout=30)
async def sync_user_data(user_id: int):
    await perform_sync(user_id)
