配置管理

配置管理

Linglong Web 提供了强大的配置系统，支持热加载。您可以定义不同环境的配置类， 并轻松地在它们之间切换。

为什么需要热加载？

在生产环境中，能够在不重启服务的情况下更新配置至关重要：

  • 零停机 - 更新设置而不中断用户
  • 动态调优 - 实时调整限流、缓存 TTL 等
  • 运维灵活性 - 修复配置错误无需重新部署

核心组件

LinglongConfigBase

提供所有设置默认值的基配置类：

from linglong_web import LinglongConfigBase

class DevConfig(LinglongConfigBase):
    DEBUG = True
    SERVICE_NAME = "my-api"
    PGSQL_HOST = "localhost"
    REDIS_HOST = "localhost"

LinglongConfig (LinglongConfigProxy)

运行时访问配置值的单例代理：

from linglong_web import LinglongConfig

# 访问配置值
service_name = LinglongConfig.SERVICE_NAME
debug_mode = LinglongConfig.DEBUG

API 参考

方法/属性 描述
init_config(config_dict, mode_name) 使用环境特定的类初始化配置
LinglongConfig.apply_updates(updates) 应用运行时配置更新（热加载）
LinglongConfig.snapshot() 获取当前配置值的快照
LinglongConfig.reset() 重置配置为初始状态
LinglongConfig.load_from_dict(dict) 从字典加载配置

配置选项

选项 类型 默认值 描述
DEBUG bool False 启用调试模式
SERVICE_NAME str "" 用于日志的服务名称
PGSQL_* str ... PostgreSQL 连接设置
REDIS_* str/int ... Redis 连接设置
CORS_ALLOWED_ORIGINS List[str] ["http://localhost"] CORS 允许的来源

使用示例

多环境配置

from linglong_web import LinglongConfigBase, init_config

class DevConfig(LinglongConfigBase):
    DEBUG = True
    PGSQL_HOST = "localhost"

class ProdConfig(LinglongConfigBase):
    DEBUG = False
    PGSQL_HOST = "postgres.internal"

# 使用环境配置初始化
init_config({
    "dev": DevConfig,
    "prod": ProdConfig,
}, mode_name="dev")

使用 YAML 进行热加载

import yaml
from linglong_web import LinglongConfig

# 从 YAML 文件加载
with open("config/prod.yaml") as f:
    settings = yaml.safe_load(f)

# 无需重启应用更新
LinglongConfig.apply_updates(settings)
