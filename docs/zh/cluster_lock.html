分布式锁

分布式锁

@cluster_lock 装饰器提供基于 Redis 的分布式锁，用于在多个实例之间协调操作。

为什么需要分布式锁？

  • 防止竞态条件 - 确保在分布式系统中单一执行
  • 资源协调 - 安全地管理共享资源
  • 幂等性 - 防止重复操作

参数

参数类型描述
keystr锁键名称
timeoutint锁超时时间（秒），默认：30

使用示例

from linglong_web.decorators import cluster_lock

@cluster_lock(key="payment:order:123")
async def process_payment(order_id: str):
    # 同一时间只有一个实例可以执行
    await payment_service.charge(order_id)
