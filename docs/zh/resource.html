资源管理

资源管理

Linglong Web 提供了统一的资源管理，用于数据库、缓存和消息队列。 它自动处理连接池、生命周期管理和优雅停机。

为什么需要统一资源管理？

在生产环境中管理多个资源连接很复杂：

  • 连接池 - 重用连接以提高性能
  • 生命周期管理 - 自动初始化和清理
  • 优雅停机 - 确保关闭期间不会丢失请求
  • 类型安全 - 完整的类型提示，支持 IDE

支持的资源

资源 库 描述
PostgreSQL asyncpg 异步 PostgreSQL 数据库驱动
Redis redis.asyncio 异步 Redis 客户端
MySQL aiomysql 异步 MySQL 数据库驱动
RabbitMQ aio-pika 异步 RabbitMQ 客户端

API 参考

函数 描述
init_resources(db_config, redis_config, mq_config) 初始化所有配置的资源
get_db() 获取数据库连接池
get_redis() 获取 Redis 连接
get_mq() 获取消息队列连接
close_resources() 关闭所有资源连接

使用示例

初始化资源

from linglong_web import init_resources, get_db, get_redis

async def init():
    await init_resources(
        db_config={
            "host": "localhost",
            "port": 5432,
            "user": "postgres",
            "password": "password",
            "database": "mydb",
        },
        redis_config={
            "host": "localhost",
            "port": 6379,
        }
    )

使用资源

from linglong_web import get_db, get_redis

async def query_users():
    db = await get_db()
    return await db.fetch("SELECT * FROM users")

async def cache_get(key: str):
    redis = await get_redis()
    return await redis.get(key)
