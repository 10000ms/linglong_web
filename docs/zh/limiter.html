限流

限流器

@limiter 装饰器为 API 端点提供基于 Redis 的限流功能。它可以防止滥用并确保公平使用资源。

为什么需要限流？

  • 防止滥用 - 阻止 DoS 攻击和过多请求
  • 公平使用 - 确保所有用户获得平等的访问权限
  • 成本控制 - 防止意外的資源消耗
  • 服务稳定性 - 保护下游服务

工作原理

  1. 使用 Redis 滑动窗口算法
  2. 按 key（IP、用户 ID 或自定义）追踪请求
  3. 超过限制时返回 429
  4. 可配置的窗口大小和最大请求数

参数

参数类型必需描述
max_callsint是窗口内允许的最大请求数
periodint是时间窗口（秒）
key_builderCallable否自定义函数来构建限流键

使用示例

from linglong_web.decorators import limiter

# 每 60 秒允许 100 个请求
@limiter(max_calls=100, period=60)
async def api_endpoint(request):
    return {"status": "ok"}
