缓存

缓存装饰器

@cacher 装饰器提供基于 Redis 的函数结果缓存。它可以通过缓存 昂贵的函数调用来显著提高性能。

为什么需要缓存？

缓存对于高性能应用至关重要：

  • 降低延迟 - 即时返回缓存结果
  • 减少负载 - 减少数据库和 API 调用
  • 节省成本 - 最小化昂贵的计算
  • 更好的用户体验 - 更快的响应时间

工作原理

装饰器的工作流程：

  1. 根据函数名和参数生成缓存键
  2. 检查 Redis 中是否存在结果
  3. 如果存在则返回缓存结果
  4. 如果不存在则执行函数并缓存结果
  5. 设置 TTL（生存时间）用于自动过期

API 参考

参数

参数 类型 必需 描述
prefix str 否 缓存键前缀（默认："cacher"）
ttl int 否 生存时间（秒），默认：300
key_builder Callable 否 自定义函数来构建缓存键

使用示例

基础用法

from linglong_web.decorators import cacher

# 缓存结果 60 秒
@cacher(ttl=60)
async def get_user(user_id: int):
    # 这将被缓存 60 秒
    return await db.fetchone(
        "SELECT * FROM users WHERE id = $1", 
        user_id
    )

使用自定义键构建器

def my_key_builder(func, *args, **kwargs):
    # 基于 user_id 和 org 自定义键
    return f"user:{args[0]}:org:{kwargs.get('org_id')}"

@cacher(ttl=300, key_builder=my_key_builder)
async def get_user_details(user_id: int, org_id: int = 1):
    return await fetch_user_details(user_id, org_id)
