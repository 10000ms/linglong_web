LinglongAppServer

服务启动器

LinglongAppServer 是核心的服务器启动类，负责处理初始化、中间件设置、 扩展钩子和优雅停机。它封装了 FastAPI，并提供了额外的企业级功能。

为什么需要 LinglongAppServer？

虽然 FastAPI 提供了优秀的异步框架，但构建生产级服务还需要额外的样板代码：

  • 设置 CORS、日志和错误处理中间件
  • 集成配置系统
  • 管理资源生命周期（数据库、Redis 等）
  • 处理优雅停机
  • 添加服务器扩展以支持自定义钩子

LinglongAppServer 将所有这些关注点封装成一个简单、统一的 API， 让您可以专注于业务逻辑。

工作原理

初始化流程

1. 创建服务器实例
2. 使用配置、路由器和扩展进行初始化
3. 服务器应用中间件（CORS、日志、错误处理）
4. 服务器注册路由
5. 服务器开始监听指定的 host/port

API 参考

LinglongAppServer 方法

方法 描述
initialize() 使用配置、路由器和扩展初始化服务器
start() 在指定的 host 和 port 上启动服务器
stop() 优雅地停止服务器
get_app() 获取底层的 FastAPI 应用实例

initialize() 参数

参数 类型 必需 描述
service_name str 是 服务名称
router APIRouter 是 包含已注册路由的 FastAPI 路由器
config_dict Dict[str, type] 是 按环境键控的配置类
extensions List[BaseServerExtension] 否 服务器扩展列表，用于钩子
title str 否 API 文档标题
version str 否 API 文档版本

使用示例

基础用法

import asyncio
from linglong_web import (
    LinglongAppServer,
    ServerRouter,
    init_config,
    init_resources,
)

async def main():
    # 初始化配置
    init_config({"dev": DevConfig}, mode_name="dev")
    
    # 初始化资源
    await init_resources(db_config={...}, redis_config={...})
    
    # 创建路由器
    router = ServerRouter()
    @router.route("/health", method="GET")
    async def health(request):
        return {"status": "ok"}
    
    # 创建并初始化服务器
    server = LinglongAppServer()
    await server.initialize(
        service_name="my-api",
        router=router.get_router(),
        config_dict={"dev": DevConfig},
    )
    
    # 启动服务器
    await server.start(host="0.0.0.0", port=8080)

asyncio.run(main())

使用服务器扩展

from linglong_web import BaseServerExtension

class MyExtension(BaseServerExtension):
    async def on_startup(self, app):
        # 服务器启动前调用
        print("服务器启动中...")
    
    async def on_shutdown(self, app):
        # 服务器停止前调用
        print("服务器停止中...")

server = LinglongAppServer()
await server.initialize(
    service_name="my-api",
    router=router.get_router(),
    config_dict={"dev": DevConfig},
    extensions=[MyExtension()],
)

优雅停机

LinglongAppServer 自动处理优雅停机：

  1. 接收 SIGTERM/SIGINT 信号
  2. 停止接受新连接
  3. 调用 on_shutdown 扩展
  4. 关闭所有资源（数据库、Redis 等）
  5. 干净地退出
