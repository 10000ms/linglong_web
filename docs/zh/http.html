HTTP 客户端

Linglong Web 提供了一个基于 httpx 增强的 HTTP 客户端，具有用于分布式追踪的 自动 request-id 传播功能。它简化了向外部服务发起 HTTP 请求的过程，同时保持了可追踪性。

为什么需要 Request-Id 传播？

在微服务架构中，跨服务边界追踪请求至关重要：

  • 分布式追踪 - 关联跨服务的日志
  • 调试 - 快速找到问题的根本原因
  • 审计追踪 - 追踪完整的请求流程

HTTP 客户端自动从传入请求中提取 request-id 并将其传播到下游服务。

核心功能

  • Request-Id 传播 - 自动传递 X-Request-Id 头
  • 连接池 - 重用连接以提高性能
  • 超时管理 - 可配置不同操作的超时时间
  • 重试逻辑 - 自动重试临时失败

API 参考

方法 描述

使用示例

基础用法

from linglong_web.http import LinglongHttpClient

client = LinglongHttpClient(base_url="https://api.example.com")

# GET 请求
response = await client.get("/users")
data = response.json()

# POST 请求
response = await client.post(
    "/users",
    json={"name": "John"}
)

配合请求上下文使用

from linglong_web.http import LinglongHttpClient
from linglong_web import RequestContext

# Request-Id 会自动传播
client = LinglongHttpClient(base_url="https://api.example.com")

async def call_service():
    request_id = RequestContext.request_id  # 自动填充
    response = await client.get("/data")
    return response.json()
