HTTP Client

HTTP Client

Linglong Web provides an enhanced HTTP client based on httpx with automatic request-id propagation for distributed tracing. It simplifies making HTTP requests to external services while maintaining traceability.

Why Request-Id Propagation?

In microservices architectures, tracking requests across service boundaries is critical:

  • Distributed Tracing - Correlate logs across services
  • Debugging - Find the root cause of issues quickly
  • Audit Trail - Track the complete request flow

The HTTP client automatically extracts the request-id from incoming requests and propagates it to downstream services.

Core Features

  • Request-Id Propagation - Automatically passes X-Request-Id header
  • Connection Pooling - Reuses connections for better performance
  • Timeout Management - Configurable timeouts for different operations
  • Retry Logic - Automatic retries for transient failures

API Reference

Method Description
get(url, **kwargs) Send GET request
post(url, **kwargs) Send POST request
put(url, **kwargs) Send PUT request
delete(url, **kwargs) Send DELETE request
patch(url, **kwargs) Send PATCH request

Usage Examples

Basic Usage

from linglong_web.http import LinglongHttpClient

client = LinglongHttpClient(base_url="https://api.example.com")

# GET request
response = await client.get("/users")
data = response.json()

# POST request
response = await client.post(
    "/users",
    json={"name": "John"}
)

With Request Context

from linglong_web.http import LinglongHttpClient
from linglong_web import RequestContext

# Request-Id is automatically propagated
client = LinglongHttpClient(base_url="https://api.example.com")

async def call_service():
    request_id = RequestContext.request_id  # Auto-populated
    response = await client.get("/data")
    return response.json()
