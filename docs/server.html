LinglongAppServer

Server

LinglongAppServer is the core server bootstrap class that handles initialization, middleware setup, extension hooks, and graceful shutdown. It wraps FastAPI and provides additional enterprise-grade features.

Why Do We Need LinglongAppServer?

While FastAPI provides an excellent async framework, building production-ready services requires additional boilerplate code:

  • Setting up CORS, logging, and error handling middleware
  • Integrating with configuration systems
  • Managing resource lifecycle (databases, Redis, etc.)
  • Handling graceful shutdown
  • Adding server extensions for custom hooks

LinglongAppServer encapsulates all these concerns into a simple, unified API, so you can focus on your business logic.

How It Works

Initialization Flow

1. Create server instance
2. Initialize with config, router, and extensions
3. Server applies middleware (CORS, logging, error handling)
4. Server registers routes from router
5. Server starts listening on specified host/port

API Reference

LinglongAppServer

Method Description
initialize() Initialize server with config, router, and extensions
start() Start the server on specified host and port
stop() Gracefully stop the server
get_app() Get the underlying FastAPI app instance

initialize() Parameters

Parameter Type Required Description
service_name str Yes Name of the service
router APIRouter Yes FastAPI router with registered routes
config_dict Dict[str, type] Yes Configuration classes keyed by environment
extensions List[BaseServerExtension] No List of server extensions for hooks
title str No API documentation title
version str No API documentation version

Usage Examples

Basic Usage

import asyncio
from linglong_web import (
    LinglongAppServer,
    ServerRouter,
    init_config,
    init_resources,
)

async def main():
    # Initialize configuration
    init_config({"dev": DevConfig}, mode_name="dev")
    
    # Initialize resources
    await init_resources(db_config={...}, redis_config={...})
    
    # Create router
    router = ServerRouter()
    @router.route("/health", method="GET")
    async def health(request):
        return {"status": "ok"}
    
    # Create and initialize server
    server = LinglongAppServer()
    await server.initialize(
        service_name="my-api",
        router=router.get_router(),
        config_dict={"dev": DevConfig},
    )
    
    # Start server
    await server.start(host="0.0.0.0", port=8080)

asyncio.run(main())

Graceful Shutdown

LinglongAppServer handles graceful shutdown automatically:

  1. Receives SIGTERM/SIGINT signal
  2. Stops accepting new connections
  3. Calls on_shutdown extensions
  4. Closes all resources (databases, Redis, etc.)
  5. Exits cleanly
