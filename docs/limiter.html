Limiter

Rate Limiter

The @limiter decorator provides Redis-based rate limiting for API endpoints. It prevents abuse and ensures fair resource usage.

Why Rate Limiting?

  • Prevent Abuse - Stop DoS attacks and excessive requests
  • Fair Usage - Ensure all users get equal access
  • Cost Control - Prevent unexpected resource consumption
  • Service Stability - Protect downstream services

How It Works

  1. Uses Redis sliding window algorithm
  2. Tracks requests per key (IP, user ID, or custom)
  3. Returns 429 when limit exceeded
  4. Configurable window size and max requests

Parameters

ParameterTypeRequiredDescription
max_callsintYesMaximum requests allowed in window
periodintYesTime window in seconds
key_builderCallableNoCustom function to build rate limit key

Usage Example

from linglong_web.decorators import limiter

# Allow 100 requests per 60 seconds
@limiter(max_calls=100, period=60)
async def api_endpoint(request):
    return {"status": "ok"}
