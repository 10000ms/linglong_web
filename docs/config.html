Config

Configuration

Linglong Web provides a powerful configuration system with hot-reload support. It allows you to define configuration classes for different environments and switch between them easily.

Why Hot-Reload?

In production environments, being able to update configuration without restarting the service is critical:

  • Zero Downtime - Update settings without interrupting users
  • Dynamic Tuning - Adjust rate limits, cache TTLs, etc. on the fly
  • Operational Flexibility - Fix configuration errors without deployment

Core Components

LinglongConfigBase

Base configuration class that provides default values for all settings:

from linglong_web import LinglongConfigBase

class DevConfig(LinglongConfigBase):
    DEBUG = True
    SERVICE_NAME = "my-api"
    PGSQL_HOST = "localhost"
    REDIS_HOST = "localhost"

LinglongConfig (LinglongConfigProxy)

Singleton proxy for accessing configuration values at runtime:

from linglong_web import LinglongConfig

# Access config values
service_name = LinglongConfig.SERVICE_NAME
debug_mode = LinglongConfig.DEBUG

API Reference

Method/Property Description
init_config(config_dict, mode_name) Initialize config with environment-specific classes
LinglongConfig.apply_updates(updates) Apply runtime config updates (hot-reload)
LinglongConfig.snapshot() Get a snapshot of current config values
LinglongConfig.reset() Reset config to initial state
LinglongConfig.load_from_dict(dict) Load config from a dictionary

Configuration Options

Option Type Default Description
DEBUG bool False Enable debug mode
SERVICE_NAME str "" Service name for logging
PGSQL_* str ... PostgreSQL connection settings
REDIS_* str/int ... Redis connection settings
CORS_ALLOWED_ORIGINS List[str] ["http://localhost"] CORS allowed origins

Usage Examples

Multi-Environment Setup

from linglong_web import LinglongConfigBase, init_config

class DevConfig(LinglongConfigBase):
    DEBUG = True
    PGSQL_HOST = "localhost"

class ProdConfig(LinglongConfigBase):
    DEBUG = False
    PGSQL_HOST = "postgres.internal"

# Initialize with environment configs
init_config({
    "dev": DevConfig,
    "prod": ProdConfig,
}, mode_name="dev")

Hot-Reload with YAML

import yaml
from linglong_web import LinglongConfig

# Load from YAML file
with open("config/prod.yaml") as f:
    settings = yaml.safe_load(f)

# Apply updates without restart
LinglongConfig.apply_updates(settings)
