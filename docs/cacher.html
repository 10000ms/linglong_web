Cacher

Caching Decorator

The @cacher decorator provides Redis-based function result caching. It can dramatically improve performance by caching expensive function calls.

Why Use Caching?

Caching is essential for high-performance applications:

  • Reduce Latency - Serve cached results instantly
  • Decrease Load - Reduce database and API calls
  • Cost Savings - Minimize expensive computations
  • Better UX - Faster response times for users

How It Works

The decorator works by:

  1. Generating a cache key from function name and arguments
  2. Checking if the result exists in Redis
  3. Returning cached result if available
  4. Executing the function and caching the result if not
  5. Setting TTL (Time To Live) for automatic expiration

API Reference

Parameters

Parameter Type Required Description
prefix str No Cache key prefix (default: "cacher")
ttl int No Time to live in seconds (default: 300)
key_builder Callable No Custom function to build cache key

Usage Examples

Basic Usage

from linglong_web.decorators import cacher

# Cache result for 60 seconds
@cacher(ttl=60)
async def get_user(user_id: int):
    # This will be cached for 60 seconds
    return await db.fetchone(
        "SELECT * FROM users WHERE id = $1", 
        user_id
    )

With Custom Key Builder

def my_key_builder(func, *args, **kwargs):
    # Custom key based on user_id and include org
    return f"user:{args[0]}:org:{kwargs.get('org_id')}"

@cacher(ttl=300, key_builder=my_key_builder)
async def get_user_details(user_id: int, org_id: int = 1):
    return await fetch_user_details(user_id, org_id)
