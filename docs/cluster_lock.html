Cluster Lock

Distributed Lock

The @cluster_lock decorator provides Redis-based distributed locking for coordinating operations across multiple instances.

Why Distributed Lock?

  • Prevent Race Conditions - Ensure single execution in distributed systems
  • Resource Coordination - Manage shared resources safely
  • Idempotency - Prevent duplicate operations

Parameters

ParameterTypeDescription
keystrLock key name
timeoutintLock timeout in seconds (default: 30)

Usage Example

from linglong_web.decorators import cluster_lock

@cluster_lock(key="payment:order:123")
async def process_payment(order_id: str):
    # Only one instance can execute this at a time
    await payment_service.charge(order_id)
