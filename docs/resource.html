Resource Management

Linglong Web provides unified resource management for databases, caches, and message queues. It handles connection pooling, lifecycle management, and graceful shutdown automatically.

Why Unified Resource Management?

Managing multiple resource connections in production environments is complex:

  • Connection Pooling - Reuse connections for better performance
  • Lifecycle Management - Automatic initialization and cleanup
  • Graceful Shutdown - Ensure no requests are lost during shutdown
  • Type Safety - Full type hints for IDE support

Supported Resources

Resource Library Description
PostgreSQL asyncpg Async PostgreSQL database driver
Redis redis.asyncio Async Redis client
MySQL aiomysql Async MySQL database driver
RabbitMQ aio-pika Async RabbitMQ client

API Reference

Function Description
init_resources(db_config, redis_config, mq_config) Initialize all configured resources
get_db() Get the database connection pool
get_redis() Get the Redis connection
get_mq() Get the message queue connection
close_resources() Close all resource connections

Usage Examples

Initializing Resources

from linglong_web import init_resources, get_db, get_redis

async def init():
    await init_resources(
        db_config={
            "host": "localhost",
            "port": 5432,
            "user": "postgres",
            "password": "password",
            "database": "mydb",
        },
        redis_config={
            "host": "localhost",
            "port": 6379,
        }
    )

Using Resources

from linglong_web import get_db, get_redis

async def query_users():
    db = await get_db()
    return await db.fetch("SELECT * FROM users")

async def cache_get(key: str):
    redis = await get_redis()
    return await redis.get(key)
